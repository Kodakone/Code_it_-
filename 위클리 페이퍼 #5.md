### 두 프레임워크를 비교하자면, 배포 / 코드의 사용성등 환경에 따라 다르다고 볼 수 있다.

PyTorch와 TensorFlow는 모두 딥러닝을 위한 프레임워크이지만, 몇 가지 주요 차이점이 있다.

PyTorch: Facebook 그룹이 제작, 연구 및 프로토타입 개발에 선호되는 편. (최근, TorchServe로 프로덕션 환경에서도 多사용)

- Python 코드와 유사. 직관적이고 간단하며, 빠른 편집을 통해 연구 개발에 유리
- 커뮤니티 지원 및 문서 정리 잘 되어 있음. 
- 동적 그래프를 사용하므로 직관적 & 유연함
- 그러나 배포가 어려움. TensorFlow에 비해 API 지원 적고, 실제 생산 환경 배포시 복잡

Tensorflow: 구글에서 제작. 안드로이드 포함 다양한 장치와 호환성으로 배포에 좋음. 

- 단일 CPU ~ 대규모 분산 시스템까지 다양한 환경 운영 가능. (대규모 Dataset 및 복잡한 모델에 적합) 
- Tensorboard로 훈련 시각화
- 간단한 모바일 지원 배포가 편함. 

- 코드 및 사용법이 복잡한 편. (단, TensorFlow 2.0 이후 Keras와 통합되어 사용감 개선)
  
결론적으로, PyTorch는 유연성과 사용성이 강점이고, TensorFlow는 배포와 확장성에서 더 유리하다고 볼 수 있다.

> # PyTorch에서 텐서란 무엇이고, NumPy의 Array(배열)과 어떤 차이가 있나요?

### Data가 딥러닝 작업에 특화되어있느냐? / 수치 작성 및 조작에 특화 되어 있느냐? 의 차이

PyTorch의 텐서는 데이터의 다차원 배열을 표현하는 데이터 구조로, NumPy의 배열과 유사한 역할.
단, PyTorch의 Tensor는 딥러닝 같은 고성능 계산에 최적화된 기능(GPU 가속 연산)을 제공한다는 점에서 NumPy 배열과 큰 차이!

1) GPU 사용
Numpy의 Array의 경우, 주로 CPU 기반의 수학, 과학 계산에 사용. GPU 활용 불가.
반면, PyTorch의 텐서는 CPU뿐만 아니라 GPU에서도 연산을 수행할 수 있다. 
GPU로 연산하려면 텐서를 .to(device) 혹은 .to('cuda') 메서드로 이동시키면 된다.
이 기능은 대규모 데이터 처리와 딥러닝 모델 학습에 유리하다.

2) 자동 미분 기능 (autograd)
또한, PyTorch 텐서는 자동 미분 기능을 지원한다. 이를 통해, 딥러닝에 필수적인 기울기 계산을 간단히 처리할 수 있음.
requires_grad=True로 설정된 텐서는 계산 그래프를 기록하며, 이를 통해 손실 함수의 경사(gradient)를 자동으로 계산한다.
반면 NumPy 배열은 이러한 기능이 없기 때문에 딥러닝 학습에는 적합하지 않다.

3) 상호 호환
또한, Pytorch의 Tensor는 Numpy의 Array와 상호 변환이 가능하다. 이를 통해 Numpy 데이터 처리 기능을 사용하면서도,
Pytorch의 고성능 연산 및 딥러닝을 사용할 수 있게 된다.

요약하자면, PyTorch 텐서는 NumPy 배열처럼 데이터 조작이 가능하지만, GPU 가속 연산과 자동 미분 기능 덕분에 딥러닝 작업에 특화된 데이터 구조이다.
